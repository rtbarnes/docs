## Commencez en trois étapes

Lancez votre site de documentation en local pour prévisualiser vos modifications. Le meilleur moyen d'y parvenir est de suivre chaque étape dans l'ordre ; bien poser les bases vous fera gagner du temps par la suite. Assurez-vous de bien comprendre chaque étape avant de passer à la suivante.

### Étape 1 : Configurez votre environnement local

<AccordionGroup>
  <Accordion title="Clonez votre dépôt en local" icon="copy">
    Lors du processus d'intégration, vous avez créé un dépôt GitHub avec le contenu de votre documentation (ou vous en aviez déjà un). Vous trouverez un lien vers ce dépôt dans votre [tableau de bord](https://dashboard.mintlify.com).

    Pour cloner le dépôt en local afin de modifier et prévisualiser votre documentation, suivez le guide [Cloner un dépôt](https://docs.github.com/fr/repositories/creating-and-managing-repositories/cloning-a-repository) de la documentation GitHub.
  </Accordion>
  <Accordion title="Lancez le serveur de prévisualisation" icon="rectangle-terminal">
    1. Installez le CLI Mintlify : `npm i -g mint`
    2. Naviguez vers votre répertoire de documentation et exécutez : `mint dev`
    3. Ouvrez `http://localhost:3000` pour voir votre documentation en direct !

    <Tip>
      La prévisualisation se met à jour automatiquement lorsque vous modifiez des fichiers.
    </Tip>
  </Accordion>
</AccordionGroup>

### Étape 2 : Déployez vos modifications

<AccordionGroup>
  <Accordion title="Installez notre application GitHub" icon="github">
    Installez l'application GitHub Mintlify depuis votre [tableau de bord](https://dashboard.mintlify.com/settings/organization/github-app).

    Notre application GitHub déploie automatiquement vos modifications sur votre site de documentation, vous n'avez donc pas à gérer les déploiements vous-même.
  </Accordion>
  <Accordion title="Personnalisez le nom et les couleurs de votre site" icon="palette">
    Pour une première modification, mettons à jour le nom et les couleurs de votre site de documentation.

    1. Ouvrez `docs.json` dans votre éditeur.
    2. Modifiez le champ `"name"` avec le nom de votre projet.
    3. Mettez à jour les `"colors"` pour correspondre à votre marque.
    4. Enregistrez et voyez vos modifications instantanément sur `http://localhost:3000`.

    <Tip>
      Essayez de changer la couleur principale pour voir la différence immédiatement !
    </Tip>
  </Accordion>
</AccordionGroup>

### Étape 3 : Mise en ligne

<Accordion title="Publiez votre documentation" icon="rocket">
  1. Validez et poussez vos modifications.
  2. Votre documentation sera mise à jour et en ligne en quelques instants !
</Accordion>
