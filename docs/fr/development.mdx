<Info>
  **Prérequis** :

  - Node.js version 19 ou supérieure
  - Un dépôt de documentation contenant un fichier `docs.json`
</Info>

Suivez ces étapes pour installer et exécuter Mintlify sur le système d'exploitation de votre choix. Nous prenons en charge Windows et macOS.

<Steps>
  <Step title="Installez le CLI Mintlify">
    ```bash
    npm i -g mint
    ```
  </Step>
  <Step title="Prévisualisez en local">
    Naviguez vers le répertoire de votre documentation, où se trouve le fichier `docs.json`, puis exécutez la commande suivante :

    ```bash
    mint dev
    ```

    Une prévisualisation locale de votre documentation sera disponible à l'adresse `http://localhost:3000`.
  </Step>
</Steps>

## Ports personnalisés

Par défaut, Mintlify utilise le port 3000. Vous pouvez personnaliser le port utilisé par Mintlify avec l'option `--port`. Par exemple, pour exécuter Mintlify sur le port 3333, utilisez la commande suivante :

```bash
mint dev --port 3333
```

Si vous essayez d'exécuter Mintlify sur un port déjà utilisé, il utilisera le prochain port disponible :

```md
Port 3000 is already in use. Trying 3001 instead.
```

## Versions de Mintlify

Veuillez noter que chaque version du CLI est associée à une version spécifique de Mintlify. Si votre prévisualisation locale ne correspond pas à la version de production, mettez à jour le CLI :

```bash
npm mint update
```

## Validation des liens

Le CLI peut vous aider à valider les liens de votre documentation. Pour identifier les liens cassés, utilisez la commande suivante :

```bash
mint broken-links
```

## Mise en forme du code

Nous vous recommandons d'utiliser des extensions sur votre IDE pour reconnaître et formater MDX. Si vous utilisez VSCode, envisagez l'[extension MDX pour VSCode](https://marketplace.visualstudio.com/items?itemName=unifiedjs.vscode-mdx) pour la coloration syntaxique, et [Prettier](https://marketplace.visualstudio.com/items?itemName=esbenp.prettier-vscode) pour la mise en forme du code.

## Dépannage

<AccordionGroup>
  <Accordion title='Erreur : Could not load the "sharp" module using the darwin-arm64 runtime'>
    Cela peut être dû à une version obsolète de node. Essayez ce qui suit :

    1. Supprimez la version actuellement installée du CLI : `npm remove -g mint`
    2. Mettez à niveau vers Node v19 ou une version supérieure.
    3. Réinstallez le CLI : `npm i -g mint`
  </Accordion>
  <Accordion title="Problème : Vous rencontrez une erreur inconnue">
    Solution : Allez à la racine de votre appareil et supprimez le dossier `~/.mintlify`. Puis exécutez à nouveau `mint dev`.
  </Accordion>
</AccordionGroup>

Curieux de savoir ce qui a changé dans la dernière version du CLI ? Consultez le [changelog du CLI](https://www.npmjs.com/package/mintlify?activeTab=versions).
