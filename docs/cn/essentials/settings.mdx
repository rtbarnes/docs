---
title: '全局设置'
description: 'Mintlify 通过 docs.json 文件让你完全控制文档的外观和体验'

---

每个 Mintlify 站点都需要一个包含核心配置项的 `docs.json` 文件。请在下方了解更多[属性](#properties)。

## 属性

<ResponseField name="name" type="string" required>
你项目的名称。用作全局标题。

示例：`mintlify`

</ResponseField>

<ResponseField name="navigation" type="Navigation[]" required>
  一个包含所有页面分组的数组
  <Expandable title="Navigation">
    <ResponseField name="group" type="string">
    分组的名称。

    示例：`Settings`

    </ResponseField>
    <ResponseField name="pages" type="string[]">
    将作为页面的 Markdown 文件的相对路径。

    示例：`["customization", "page"]`

    </ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="logo" type="string or object">
  Logo 图片的路径，或者包含 "light" 和 "dark" 模式下 logo 图片路径的对象
  <Expandable title="Logo">
    <ResponseField name="light" type="string">
      浅色模式下 logo 的路径
    </ResponseField>
    <ResponseField name="dark" type="string">
      深色模式下 logo 的路径
    </ResponseField>
    <ResponseField name="href" type="string" default="/">
      点击 logo 后跳转到的链接
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="favicon" type="string">
  favicon 图片的路径
</ResponseField>

<ResponseField name="colors" type="Colors">
  全局主题的十六进制颜色代码
  <Expandable title="Colors">
    <ResponseField name="primary" type="string" required>
      主色。最常用于浅色模式下的高亮内容、章节标题和强调色
    </ResponseField>
    <ResponseField name="light" type="string">
      深色模式下的主色。最常用于深色模式下的高亮内容、章节标题和强调色
    </ResponseField>
    <ResponseField name="dark" type="string">
      用于重要按钮的主色
    </ResponseField>
    <ResponseField name="background" type="object">
      浅色和深色模式下的背景颜色
      <Expandable title="Object">
        <ResponseField name="light" type="string" required>
          浅色模式下背景的十六进制颜色代码
        </ResponseField>
        <ResponseField name="dark" type="string" required>
          深色模式下背景的十六进制颜色代码
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="topbarLinks" type="TopbarLink[]">
  你希望在顶部栏中包含的链接的 `name` 和 `url` 数组
  <Expandable title="TopbarLink">
    <ResponseField name="name" type="string">
    按钮的名称。

    示例：`Contact us`
    </ResponseField>
    <ResponseField name="url" type="string">
    点击按钮后跳转的 URL。示例：`https://mintlify.com/docs`
    </ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="topbarCtaButton" type="Call to Action">
  <Expandable title="Topbar Call to Action">
    <ResponseField name="type" type={'"link" or "github"'} default="link">
    Link 会显示一个按钮。GitHub 会显示提供的 URL 所对应仓库的信息，包括 GitHub Star 数量。
    </ResponseField>
    <ResponseField name="url" type="string">
    如果是 `link`：按钮跳转到的链接。
    
    如果是 `github`：用于加载 GitHub 信息的仓库链接。
    </ResponseField>
    <ResponseField name="name" type="string">
    按钮内的文本。仅当 `type` 为 `link` 时必填。
    </ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="versions" type="string[]">
  版本名称数组。仅当你希望在导航栏中通过下拉菜单显示不同版本的文档时使用。
</ResponseField>

<ResponseField name="anchors" type="Anchor[]">
  anchor 数组，包含 `icon`、`color` 和 `url`。
  <Expandable title="Anchor">
    <ResponseField name="icon" type="string">
    用于标识 anchor 的 [Font Awesome](https://fontawesome.com/search?q=heart) 图标。

    示例：`comments`
    </ResponseField>
    <ResponseField name="name" type="string">
    anchor 标签的名称。

    示例：`Community`
    </ResponseField>
    <ResponseField name="url" type="string">
      表明哪些页面属于该 anchor 的 URL 起始路径。通常这是你放置页面的文件夹名称。
    </ResponseField>
    <ResponseField name="color" type="string">
      anchor 图标背景的十六进制颜色。也可以传入一个包含 `from` 和 `to` 属性的对象（两者均为十六进制颜色）来形成渐变色。
    </ResponseField>
    <ResponseField name="version" type="string">
      当你希望在选择对应文档版本之前隐藏某个 anchor 时使用。
    </ResponseField>
    <ResponseField name="isDefaultHidden" type="boolean" default="false">
      如果你希望在直接链接他人进入该 anchor 内文档之前将其隐藏，请传入 `true`。
    </ResponseField>
    <ResponseField name="iconType" default="duotone" type="string">
      可选值之一："brands"、"duotone"、"light"、"sharp-solid"、"solid" 或 "thin"
    </ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="topAnchor" type="Object">
  覆盖最顶部 anchor 的默认配置。
  <Expandable title="Object">
    <ResponseField name="name" default="Documentation" type="string">
      最顶部 anchor 的名称
    </ResponseField>
    <ResponseField name="icon" default="book-open" type="string">
      Font Awesome 图标。
    </ResponseField>
    <ResponseField name="iconType" default="duotone" type="string">
      可选值之一："brands"、"duotone"、"light"、"sharp-solid"、"solid" 或 "thin"
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="tabs" type="Tabs[]">
  导航标签数组。
  <Expandable title="Tabs">
    <ResponseField name="name" type="string">
      标签名称。
    </ResponseField>
    <ResponseField name="url" type="string">
      表明哪些页面属于该标签的 URL 起始路径。通常这是你放置页面的文件夹名称。
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="api" type="API">
  API 设置的配置。可在 [API 组件](/docs/cn/api-playground/demo) 中了解更多关于 API 页面的内容。
  <Expandable title="API">
    <ResponseField name="baseUrl" type="string">
      所有 API 端点的基础 URL。如果 `baseUrl` 是一个数组，将启用多个可供用户切换的基础 URL 选项。
    </ResponseField>

    <ResponseField name="auth" type="Auth">
      <Expandable title="Auth">
        <ResponseField name="method" type='"bearer" | "basic" | "key"'>
          用于所有 API 端点的身份验证策略。
        </ResponseField>
        <ResponseField name="name" type="string">
        API playground 中使用的身份验证参数名称。

        如果 method 为 `basic`，格式应为 `[usernameName]:[passwordName]`
        </ResponseField>
        <ResponseField name="inputPrefix" type="string">
        作为身份验证输入字段前缀的默认值。

        例如，若 `inputPrefix` 为 `AuthKey`，则身份验证字段的默认输入结果将继承为 `AuthKey`。
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="playground" type="Playground">
      API playground 的配置

      <Expandable title="Playground">
        <ResponseField name="mode" default="show" type='"show" | "simple" | "hide"'>
          playground 是显示、隐藏，还是仅展示端点而不提供用户交互（`simple`）。

          更多内容请参阅 [playground 指南](/docs/cn/api-playground/demo)
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="maintainOrder" type="boolean">
      启用此标志可确保 OpenAPI 页面中的键顺序与 OpenAPI 文件中定义的键顺序保持一致。

      <Warning>此行为很快将默认启用，届时该字段将被弃用。</Warning>
    </ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="openapi" type="string | string[]">
  指向你的 OpenAPI 文件的 URL 或相对路径的字符串，或字符串数组。
  
  示例：
  <CodeGroup>
    ```json Absolute
    "openapi": "https://example.com/openapi.json"
    ```
    ```json Relative
    "openapi": "/openapi.json"
    ```
    ```json Multiple
    "openapi": ["https://example.com/openapi1.json", "/openapi2.json", "/openapi3.json"]
    ```
  </CodeGroup>

</ResponseField>

<ResponseField name="footerSocials" type="FooterSocials">
  社交媒体账号对象，其中键值对分别表示社交媒体平台及其账号 URL。
  
  示例：
  ```json
  {
    "x": "https://x.com/mintlify",
    "website": "https://mintlify.com"
  }
  ```
  <Expandable title="FooterSocials">
    <ResponseField name="[key]" type="string">
    以下值之一：`website`、`facebook`、`x`、`discord`、`slack`、`github`、`linkedin`、`instagram`、`hacker-news`
    
    示例：`x`
    </ResponseField>
    <ResponseField name="property" type="string">
    社交平台的 URL。
    
    示例：`https://x.com/mintlify`
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="feedback" type="Feedback">
  启用反馈按钮的配置

  <Expandable title="Feedback">
    <ResponseField name="suggestEdit" type="boolean" default="false">
    启用一个按钮，允许用户通过 pull request 提交编辑建议
    </ResponseField>
    <ResponseField name="raiseIssue" type="boolean" default="false">
    启用一个按钮，允许用户对文档提出 issue
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="modeToggle" type="ModeToggle">
  自定义深色模式切换。
  <Expandable title="ModeToggle">
    <ResponseField name="default" type={'"light" or "dark"'}>
      如果你希望对新用户始终显示浅色或深色模式，请进行设置。未设置时，我们默认采用与用户操作系统相同的模式。
    </ResponseField>
    <ResponseField name="isHidden" type="boolean" default="false">
      设置为 true 可隐藏深色/浅色模式切换。你可以将 `isHidden` 与 `default` 结合使用，以强制让文档仅使用浅色或深色模式。例如：
      
      <CodeGroup>
      ```json Only Dark Mode
      "modeToggle": {
        "default": "dark",
        "isHidden": true
      }
      ```

      ```json Only Light Mode
      "modeToggle": {
        "default": "light",
        "isHidden": true
      }
      ```
      </CodeGroup>

    </ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="backgroundImage" type="string">
  在每个页面背后显示的背景图片。可参考 [Infisical](https://infisical.com/docs) 和 [FRPC](https://frpc.io) 的示例。
</ResponseField>
